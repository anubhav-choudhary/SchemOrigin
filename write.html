How to write ER Model Description (ERD)

Let us write ER Model Description (ERD) for given Video Library Shop ER Model. Following are the steps to do the same.

  1. Make list of words written in rectangle : In this step we will make list of words written in rectangle and the words will be prefixed by “#” sign. In this case following is the required list.

    #Customer
    #RegisteredMembers
    #NonRegisteredMembers
    #VCD
    #AvailableVCD
    #NewVCD
    #Shop

  2. Add attributes to each Entity Set : In this step we will add attributes connected to each entity set. We will write all attributes of entity set separated by comma. If an attributed is key then “_pk” is suffixed with the attribute and if attribute is multi valued then “_mv” is suffixed. After listing all attributes semicolon is added at the last. In this case following is the required list

    #Customer:memberid_pk,pwd;
    #RegisteredMembers:address,name,age;
    #NonRegisteredMembers:limit;
    #VCD:vid_pk,type,title,status,price;
    #AvailableVCD;
    #NewVCD;
    #Shop:shop_id,shopname;

  3. Add “%%” after the above code to start entity set relation section : The sign “%%” indicates the start of entity set relation section. So after adding “%%”, the given code becomes

    #Customer:memberid_pk,pwd;
    #RegisteredMembers:address,name,age;
    #NonRegisteredMembers:limit;
    #VCD:vid_pk,type,title,status,price;
    #AvailableVCD;
    #NewVCD;
    #Shop:shop_id,shopname;
    %%

  4. Identify “ISA” relationships i.e Generalization/Specialization : In this step for each “ISA” relation identify general and special entity sets and add relationship code using the following syntax

    Syntax : @#GeneralEntity:ISA:#Special1,#Special2,#Special3;

    Using the above syntax, the given code becomes

    #Customer:memberid_pk,pwd;
    #RegisteredMembers:address,name,age;
    #NonRegisteredMembers:limit;
    #VCD:vid_pk,type,title,status,price;
    #AvailableVCD;
    #NewVCD;
    #Shop:shop_id,shopname;
    %%
    @    #Customer:ISA:#RegisteredMembers,#NonRegisteredMembers;
    @#VCD:ISA:#AvailableVCD,#NewVCD;

  5. Identify the “Diamonds” i,e Entity Relationship : In this step relationship between two entity sets are identified and relationship code is added using the following syntax

    Syntax : @RelationName:DescriptiveAttribute1,DescriptiveAttribute2:#GeneralEntity:X:X:#Special1;
    Here X could be 1 or M to show the cardinality of the relationship.

    Using the above syntax, the given code becomes

    #Customer:memberid_pk,pwd;
    #RegisteredMembers:address,name,age;
    #NonRegisteredMembers:limit;
    #VCD:vid_pk,type,title,status,price;
    #AvailableVCD;
    #NewVCD;
    #Shop:shop_id,shopname;
    %%
    @    #Customer:ISA:#RegisteredMembers,#NonRegisteredMembers;
    @#VCD:ISA:#AvailableVCD,#NewVCD;
    @Purchases:#Customer:1:M:#AvailableVCD;
    @Sees:#Customer:M:M:#AvailableVCD;
    @Rents:rentdate:#RegisteredMembers:M:M:#VCD;
    @Keeps:#Shop:1:M:#AvailableVCD;
    @Orders:#Shop:1:M:#NewVCD;

The code illustrated above is the required ER Model Description (ERD).

About SchemOrigin


In this project, ER Model Description Script (ERD) is introduced along with its compiler. ER Model Description Script (ERD) is used to represent graphical ER model to its textual equivalent. Its compiler translates given ER Model Description Script (ERD) to its equivalent Relational Database schema.

Features of Schema Generator

  1. Converts ER Model to Database Schema.
  2. It can apply 1-1, 1-M, M-N and ISA relationship to entities.
  3. Multi-Valued attributes are supported.
  4. Descriptive attributes are supported.
  5. Automatic attribute name conflict resolution.
  6. Syntax and semantic errors can be detected in ER Model Description script (ERD).
