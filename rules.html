Rules for ER Model Description (ERD)

ER Description Script (ERD) has two sections separated by section separator "%%" namely "Entity Set Definition Section" and "Entity Set Relation Section". In entity set definition section all the entity sets are defined along with their attributes while in entity set relation section relation between entity sets are defined.

[Entity Set Definition Section]
%%
[Entity Set Relation Section]

Entity Name : Entity name starts with '#' sign. The next character after # sign must be an alphabet after that numbers and underscore "_" are also allowed. Examples

#MyEntity
#Entity1
#Entity_1

Attribute Name : Name of attribute starts with only alphabets but it can have numbers and underscore "_" sign as well from second postion.

If attribute is key "_pk" is added as suffix. for example

rollno_pk

If attribute is multivalued _mv is added as suffix. for example

phoneno_mv

Entity Definition : To define an entity, entity name is given followed by attribute name. Following is the syntax of the same

Syntax : #ENTITY_NAME:attribute1,arrribute2,attribute3;

Entity Relation : Relations between entities can be defined using following syntax. Note that in this section every "ISA" relationship must be defined before other relationships.

Syntax of Generalization/Specialization : @#GeneralEntity:ISA:#Special1,#Special2,#Special3;

Syntax of Binary Relationship : @RelationName:DescriptiveAttribute1,DescriptiveAttribute2:#GeneralEntity:X:X:#Special1;
Here X could be 1 or M to show the cardinality of the relationship.

About SchemOrigin


In this project, ER Model Description Script (ERD) is introduced along with its compiler. ER Model Description Script (ERD) is used to represent graphical ER model to its textual equivalent. Its compiler translates given ER Model Description Script (ERD) to its equivalent Relational Database schema.

Features of Schema Generator

  1. Converts ER Model to Database Schema.
  2. It can apply 1-1, 1-M, M-N and ISA relationship to entities.
  3. Multi-Valued attributes are supported.
  4. Descriptive attributes are supported.
  5. Automatic attribute name conflict resolution.
  6. Syntax and semantic errors can be detected in ER Model Description script (ERD).
